💡 Refactoring UI の重要ポイント
1. 余白（Spacing）は UI を劇的に良くする

例えば：

要素間のスペースは均等にする

4, 8, 12, 16... のようにスケール化する

狭い場所では余白を詰めず、むしろ広げる

余白の原則ひとつで、B*Nデザインが「プロっぽく」なる。

2. コントラストは強く、曖昧にしない

グレーは薄すぎると読みにくい

ボタンは背景と明確に差をつける

影は弱くしすぎず、強すぎず「意味を持たせる」

コントラストが弱い UI は、プロダクトが“安っぽく”見える。

3. 色は少なく・役割を明確に

使う色は 2〜3 色に絞る

色には役割（注意/成功/警告）を持たせる

彩度を上げすぎない

特に「アクセントカラーを使いすぎるな」というのが重要。

4. 視覚的階層を意識して並べる

UI では「どれが重要か」が一瞬でわかる必要がある。

重要 → 太字・大きめ・濃い色

補足 → 小さい・薄め

関連する項目は近づける

関係ない項目は離す

5. シャドウ・角丸・境界線を“意味のある”ものにする

影は深さを示すためにつける

角丸は階層・安全性を表現

境界線の代わりに影・余白・背景色の違いを使う

6. 一貫性を保つ

最も UI が貧弱になる理由は「一貫性がない」こと。

同じ意味の要素は同じスタイル

余白スケールを統一

色の使い方を統一

タイポグラフィ階層を統一

これらが徹底されるだけで UI の完成度は爆上がりする。

📘 Refactoring UI の特に有名な教え
◾ Icon は“outline”なら全て outline

mix しない。

◾ ボタンは「丸い or 角ばった」どちらかへ寄せる

中途半端に混ざると素人感が出る。

◾ 「線」より「影」「余白」で区切る

境界線を多用しない。

◾ タイポグラフィにはテンプレがある
