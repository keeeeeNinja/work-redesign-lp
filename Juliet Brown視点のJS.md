0. コンセプトまとめ（Juliet 的な前提）

Juliet Brown の考え方をあなたのサイトに当てはめると、アニメーションの目的は：

メッセージを直感的に伝えること

「Web Developer & Writer です」

「実績はちゃんと公開・出版されたものです」

「落ち着いた信頼感と人柄がある人です」

ストーリーの流れをわかりやすくすること

冒頭：自己紹介（あなたという人物）

中盤：代表作紹介（実績）

詳細：個別プロジェクトのケーススタディ

ユーザーの感情を軽く動かすこと

ほんの少しワクワク

読みやすさによる安心感

猫の写真でクスッとした親近感

つまり

「派手なエフェクト」ではなく、
“目線誘導＋感情のニュアンス” をつくる動き
を JS で足していきます。

1. サイト全体に共通するモーション設計
1-1. モーションのトーン

速すぎず・派手すぎない、上品でやわらかい動き

主に扱うプロパティ：

opacity

transform: translateY / translateX / scale

box-shadow

イージング：power3.out や expo.out のような、
立ち上がりは素早く・終わりはゆっくり の曲線

1アニメーションの時間：0.4〜0.8秒が中心

1-2. 技術スタックの前提

スクロールやセクション単位の演出 → GSAP + ScrollTrigger

ホバー・クリックなどインタラクティブな部分 → 素の JS or GSAP

ページ遷移演出（あれば） → 余裕があれば barba.js や Next のレイアウトトランジション

※ Three.js は今回のデザインだと「必須ではない」ので、
　まずは GSAP で 2D の動きだけで世界観を完成させる想定にしておきます。

1-3. アクセシビリティ & パフォーマンス

prefers-reduced-motion: reduce を検出して、
その場合は 初期状態＝アニメ後の状態 にしてアニメ処理を一切走らせない。

すべて transform / opacity ベースで、レイアウト再計算が起きないようにする。

スマホでは

同時に動く要素を減らす

スクロールトリガーを少なめに

2. About ページ（1枚目）の演出案
2-1. ヒーローコピーの「ストーリー導入」

目的：
「この人は何者か？」を、落ち着いた動きで印象づける。

動き案：

ページロード時：

H1「Webサイトのコーディングと〜」
→ ほんの少し下からふわっと（opacity 0 → 1, translateY 20px → 0）

2〜3文の本文
→ H1 より 0.1〜0.2秒遅らせて、同じモーションで順番に出す（stagger）

HTML/CSS/JavaScript の太字部分だけ、

最初の表示から 1秒後くらいに、色を少しだけ強くする or 下線がスッと伸びる
→ 「自分の専門はここです」という視線誘導になる。

Juliet 的ポイント：
「私は〇〇ができます」というメッセージを視覚的に補強するだけで、
単なるカッコつけのアニメーションにしない。

2-2. プロフィールカード＋猫写真の“人柄セクション”

目的：
「どんな人なのか」を柔らかく伝える。
猫が“キャラクター”になっているので、ここは感情重視。

動き案：

スクロールでプロフィールカードがビューポートに入るタイミングで：

カード全体が、わずかに拡大しながら浮き上がるように表示
（opacity 0 → 1, translateY 40px → 0, scale 0.98 → 1）

カード内の項目（趣味・好きな食べ物・言語・AI）は、上から順番に 0.05〜0.1秒刻みでフェードイン（stagger）

猫写真：

カードと同タイミングか、0.2秒遅らせてフェードイン

hover 時にだけ、scale 1 → 1.02 / translateY 0 → -4px 程度で
ふんわり持ち上がるような動き

その際、下のキャプション Working Buddy の不透明度を少し上げる

Juliet 的ポイント：
「猫がただの飾りではなく、あなたの“相棒”である」ことを
小さなモーションで感じ取らせる。

2-3. 下部 Works ティーザーの“次への橋渡し”

目的：
「この先に実績があるよ」というストーリーの流れを強調する。

動き案：

WORKS 見出しの上のラベル（小文字）：

スクロールで見えたときに、左から右へ細線がスッと伸びる

3つの作品カード（秋のスイーツ／ラストダイエット／ポートフォリオ）：

それぞれ左→右に少しスライド＆フェードイン
（stagger 0.15〜0.2秒）

hover で：

画像がわずかにズームイン（scale 1.03 程度）

Explore → の矢印部分だけ 2〜3px 右にスライド

Juliet 的ポイント：
「あなたの目線を、自己紹介から作品へ自然に誘導する」ためのアニメ。
「続きがある」というストーリーの“予告編”の役割。

3. Works 一覧ページ（2枚目）の演出案
3-1. ページ冒頭のコピー

目的：
「ここは作品ギャラリーですよ」と一瞬で理解してもらう。

動き案：

文章の中のキーワード Webサイト と Kindle を
最初の表示から 1秒後に、色 or ウェイトだけふんわり変える。
→ 「自分の活動領域」を暗いハイライトで見せる感じ。

Juliet なら、ここもアニメーションしすぎず、
“ほぼ静止画の中のワンポイント” ぐらいに抑えるはず。

3-2. 各作品カード（横並びレイアウト）のストーリー

ここがこのページの本丸です。

目的：
それぞれの作品を“シーン”として見せる。
実績ページが「ケーススタディの一覧」に見えるようにする。

動き案：

スクロールでカードが入ってきたタイミングで：

左のテキストブロック：左から薄くスライドイン＆フェード

右の画像：少し遅れて、下から上へふわっと出てくる
（テキスト → 画像の順で 0.15秒ぐらいディレイ）

各カード内の構造：

WEBSITE / BOOKS のラベル：
カードの登場時に、下線が伸びる or 不透明度が上がる。

タイトル：テキストだけ少し遅れてフェードイン

説明文：さらに 0.05〜0.1秒遅れて

hover 時：

カード全体が 1〜2px 持ち上がって影が少しだけ強くなる

Explore → ボタンの背景色がじわっと変わる（0.2秒程度）

Juliet 的ポイント：

スクロールの流れの中で、
「1カード = 1章」 という読みやすいリズムを作る。

テキスト → 画像の順に出すことで、
「タイトルを認識 → ビジュアルで理解」という順番をサポート。

4. プロジェクト詳細ページ（3枚目）の演出案

ここは「ケーススタディのストーリー」を見せる場所なので、
Juliet なら最も物語的なモーションを入れてくる箇所だと思います。

4-1. ヒーロー部分（タイトル＋概要）

目的：
このページが「秋の人気スイーツ特集」のストーリーページであると印象付ける。

動き案：

WEBSITE ラベル → 左から線が伸びて出現

「秋の人気スイーツ特集」タイトル → 下からふわっと

説明文 → タイトルより 0.1秒遅れて

すぐ下の大きなメインビジュアルカード（栗のイメージ）：

タイトル・サブコピー → 先に出す

2枚の写真（栗アップ／森の写真）は、横方向に少しずつずらしながらフェードイン

ほんのわずかにパララックス（スクロールに応じて 2〜3px 差をつけて動く）
→ 「奥行き」と「秋の世界に入る感じ」が出る。

4-2. 中盤のセクション（PC版レイアウトの紹介）

目的：
コンテンツの構造やレイアウトの工夫を、視覚的に理解してもらう。

動き案：

上から順にカードが入ってくるときに：

見出し → スーッと表示

内側のテキスト → 少し遅れて

画像群 → さらに遅れて（中のコンテンツを1つずつ軽く stagger）

各カードの hover：

影を強く、背景をほんの少しだけ明るくして
「ここに目を向けてね」というサインにする。

Juliet 的には、ここは「レイアウトの理解を助ける動き」なので、
スクロールに合わせた“ページ送り”的なモーションになるはずです。

4-3. スマートフォン版レイアウトのセクション

目的：
レスポンシブへのこだわりを伝える。

動き案：

スマホ3画面の画像を、

中央の1枚を基準に、左右の2枚が少し遅れてスライドインする演出

軽めの浮遊感（hover or スクロールで 1〜2px 上下移動）

Juliet 的ポイント：
「レスポンシブです」とテキストで語るだけでなく、
“複数画面が並んでいる・動いている”ことで体感させる。

4-4. フッター前の「他の制作物」セクション

目的：
ユーザーに次のストーリーへの出口を提示する。

動き案：

セクションのタイトル「他の制作物」：

画面の下からスライドイン（スクロール時）

3つの作品カード：

About ページと同様に横方向から stagger 表示

hover でサムネイルが軽くズーム

→ ここで「このページを読み終えたら、次はどれを見てみよう？」という
ストーリーのループを作るイメージ。

5. 共通コンポーネントのインタラクション
5-1. ヘッダー（既にいい感じなので、ほんの微調整）

ページロード時：

左のロゴ（アイコン＋名前＋肩書き）は静止でよい

右の Home / About / Contact は 0.1秒ずつ stagger フェードイン

スクロールダウンでヘッダーを少しだけ縮める（高さ・影）

上方向のモーションで「固定ヘッダー」として存在を控えめに

ナビ hover：

下線が左から右へ 0.15秒でスライド

Contact ボタン：

hover で うっすら背景色がつき、影がごく僅かに出る

Juliet 的ポイント：
ナビは「アニメーションの主役」ではないので、
ユーザーの集中を奪わない程度の小さな変化で十分。

5-2. 右側の丸いボタン（セクションナビ？）

スクリーンショットの右側にある黒い丸ボタン（3本線アイコン）は、
セクションナビ or トップへ戻るボタンのように見えます。

ここはインタラクション次第で、サイトの“個性”になるポイントです。

動き案：

スクロール位置に応じて

上：薄い

中央：はっきり

下：薄い
のような不透明度変化 or スケール変化をつける。
→ 今どのエリアにいるかの“感覚的インジケータ”になる。

hover すると、中の3本線が少しふにゃっと変形 or 間隔が広がる。

クリックで

サイドメニュー or スクロール位置一覧を滑らかに表示。

Juliet 的には、「この UI で何をさせたいか」を明確にしたうえで、
その目的をサポートする動きを付けるはずです。

6. 実装フェーズに入るための ToDo リスト

アニメーションマップを作る

今書いた案から「絶対やりたいもの」と「後回しでも良いもの」を色分け

まずは About ヒーロー + Works カード + プロジェクト詳細ヒーロー くらいが第一弾

モーションの共通パラメータを決める

duration, easing, translate距離, scale などの“モーションデザインシステム”

例：

duration: 0.5 / 0.7 / 1.0s の3段階

distance: 20 / 40px

ease: power3.out で統一

GSAP の初期セットアップ

ScrollTrigger を読み込み

各セクションに data-animate="hero", data-animate="card" などの属性をつける

それをループで拾ってアニメを当てていく

prefers-reduced-motion 対応

JS の最初でフラグを立てて、
if (prefersReducedMotion) return; で全アニメ処理を抜ける

PC → SP での確認

モーションがSPでくどく感じたら

距離を半分

一部アニメを無効化
などの調整を入れる
