サマラ風ワイヤーフレーム案（現状LPからの置き換え）

目的
- SaaS系の情報設計と視覚リズムを取り入れて「価値の即時理解」と「信頼の積み上げ」を強化する。

ページ構成（上から順）

【ナビ（簡易）】
目的: 迷わない導線とページの信頼感
- 左: サイト名 / 小さなロゴ
- 右: セクションリンク（課題 / 流れ / 実例 / 相談）+ CTA

【Hero｜価値＋デモ】
目的: 一読で「誰の何をどう変えるか」を伝える
- 左: H1 + サブコピー + 主要CTA + 小さな補足
- 右: 視覚要素（再設計フローの簡略図 or 仕事棚卸しのUI風カード）
使う素材: `LP設計案/03_再設計フロー.svg` を簡略表示
既存からの流用: index.html の hero 文

【ロゴ/信頼の帯（簡易）】
目的: “見た瞬間の安心感”を作る
- 「支援領域」や「対象業種」などのタグ群（ロゴが無い場合の代替）
- 例: 事務 / 営業 / 企画 / 管理職 など

【Value Props｜3〜4カード】
目的: 一瞬で理解できる「得られること」
- 3〜4枚のカード（見出し + 1文）
既存からの流用: value セクション

【Problem to Insight｜共通課題の可視化】
目的: 課題の共感と「努力不足ではない」メッセージの強調
- 左: 課題リスト
- 右: 洞察の短文（努力の向きと配分の再設計）
既存からの流用: summary セクション

【How it works｜再設計プロセス】
目的: 抽象概念を具体化する
- 横並び or ステップカード（0〜7を4〜5枚に整理）
- 右端/下部に「伴走があると精度が上がる」補足
既存からの流用: flow セクション

【Feature Highlight｜左右交互レイアウト×2〜3】
目的: “具体的に何をするか”を視覚で説明
- 例1: 「棚卸し」+ カードUI
- 例2: 「4分類」+ 図解
- 例3: 「集中枠」+ 週間スケジュール風
既存からの流用: flow + value の要素を分割再配置

【Proof / Results｜事例のミニカード】
目的: 変化の証拠を提示
- 3カード（Before / 再設計 / After を短く）
既存からの流用: cases セクション（短縮）

【Who it’s for｜向き・不向き】
目的: ミスマッチを防ぎ信頼を上げる
- 2カラムのリスト
既存からの流用: fit セクション

【FAQ｜よくある質問】
目的: 不安の解消
- 4〜6問（料金 / 期間 / 相談内容 / 勧誘有無など）
既存からの流用: 今は無いので新規作成

【CTAバナー｜相談誘導】
目的: 1ページ内の再アクション
- 背景付きの横長バナー
- CTAボタン + 小さな注意書き
既存からの流用: consultation セクション要約

【相談フォーム｜詳細説明】
目的: 具体的な次の行動を明確化
- 左: 相談内容の説明 + チェックリスト
- 右: フォーム埋め込み or 外部リンク
既存からの流用: consultation セクション

【Footer｜一言】
目的: メッセージの収束
既存からの流用: footer 文

視覚コンセプト（サマラ寄せの要点）
- 2カラム、カード、グリッドで情報の塊を作る
- セクション背景を交互に変えて視線のリズムを作る
- UI風の図や簡単な図解を使い「再設計」を具体化
- タイポの階層差を明確にして“見出しだけでも流れが分かる”構成にする

次のステップ（コーディング手順の準備）
1) この構成に合わせて既存の見出し/本文を短文化
2) セクション単位でHTMLの骨組み（header/section/figure/ul）を作成
3) CSSでレイアウト（2カラム/カード/余白）を先に完成
4) 背景/装飾/図解などの視覚要素を追加
5) CTA/フォーム導線を最後に調整
